# QUALITY POLICY

This system enforces production-grade code quality.

---

## ACCEPTANCE THRESHOLD

- Minimum score: 9/10
- No critical issues allowed
- No unhandled edge cases allowed

---

## REJECTION RULES

Reject if:
- missing validation
- unsafe input handling
- unclear ownership of logic
- potential runtime errors
- missing authorization checks

---

## LOOP LIMIT

Maximum 2 iterations per task.

After that → escalate to human.
